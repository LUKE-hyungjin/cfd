# 연속 방정식

{{% hint info %}}
**선수지식**: [유체란 무엇인가](/cfd/ko/docs/fundamentals/fluid-mechanics/what-is-fluid)
{{% /hint %}}

## 한 줄 요약
> **들어온 만큼 나간다. 유체는 갑자기 생기거나 사라지지 않는다.**

## 왜 필요한가?

유체 시뮬레이션의 첫 번째 법칙입니다. **질량 보존** — 물리학의 가장 기본적인 원리죠.

호스로 물을 뿌릴 때, 호스 입구로 들어간 물의 양 = 출구로 나온 물의 양입니다. 당연하죠? 물이 호스 안에서 사라지거나 갑자기 생기진 않으니까요.

이 "당연한 것"을 수식으로 쓴 게 연속 방정식입니다.

## 직관적 이해

### 비유: 고속도로 톨게이트

고속도로에 차가 흐른다고 생각해봅시다:

- 톨게이트 A로 시간당 100대 진입
- 중간에 차가 사라지거나 생기지 않음
- 톨게이트 B로 시간당 100대 나감

**들어온 차 = 나간 차**. 이게 연속 방정식의 핵심입니다.

### 호스 실험

굵은 호스와 가는 호스를 연결했을 때:

```
굵은 부분        가는 부분
  =====>          =>
  느림            빠름
```

- 굵은 곳: 단면적 크고, 속도 느림
- 가는 곳: 단면적 작고, 속도 빠름

왜? **같은 양의 물**이 좁은 곳을 통과하려면 빨리 가야 하니까!

## 수식

### 기본형 (비압축성 유체)

$$
\nabla \cdot \mathbf{u} = 0
$$

**각 기호의 의미:**
- $\nabla \cdot$ : 발산(divergence) — "퍼져나가는 정도"
- $\mathbf{u}$ : 속도 벡터 (m/s) — 유체가 움직이는 방향과 빠르기

**해석**: 어떤 점에서 유체가 "퍼져나가는 양" = 0. 즉, 들어오는 양과 나가는 양이 같다.

### 성분으로 풀어쓰기

3차원에서:

$$
\frac{\partial u}{\partial x} + \frac{\partial v}{\partial y} + \frac{\partial w}{\partial z} = 0
$$

**각 기호의 의미:**
- $u$ : x방향 속도
- $v$ : y방향 속도
- $w$ : z방향 속도
- $\frac{\partial u}{\partial x}$ : x방향으로 갈 때 u가 얼마나 변하나

**해석**: 각 방향으로 속도가 변하는 정도를 다 더하면 0.

### 적분형 (더 직관적)

$$
\oint_S \mathbf{u} \cdot \mathbf{n} \, dA = 0
$$

**해석**: 어떤 공간을 감싸는 표면을 통해 들어오는 유체 = 나가는 유체

## 실제 예시

### 혈관 분기점

대동맥에서 두 갈래로 나뉠 때:

```
       Q₁ (왼쪽으로)
      ↗
Q₀ →
      ↘
       Q₂ (오른쪽으로)
```

연속 방정식:
$$
Q_0 = Q_1 + Q_2
$$

- $Q$ : 유량 (단위 시간당 지나가는 부피, m³/s)

**의미**: 들어온 피(Q₀) = 왼쪽으로 간 피(Q₁) + 오른쪽으로 간 피(Q₂)

### 협착 (Stenosis)

혈관이 좁아진 부분:

```
정상 부분      협착 부분      정상 부분
  ====>          =>          ====>
  느림          빠름          느림
```

$$
A_1 v_1 = A_2 v_2
$$

- $A$ : 단면적
- $v$ : 속도

**의미**: 단면적 × 속도 = 일정. 좁아지면 빨라지고, 넓어지면 느려진다.

협착 부분에서 혈류가 빨라지는 이유가 바로 이것!

## 비압축성 가정

위의 수식들은 **비압축성 유체**를 가정합니다.

| 가정 | 의미 | 적용 |
|------|------|------|
| 비압축성 | 밀도가 일정 ($\rho$ = 상수) | 물, 혈액 |
| 압축성 | 밀도가 변함 | 공기 (고속) |

**생체유체 CFD에서**: 혈액은 비압축성으로 봅니다. 밀도가 거의 안 변하니까요.

## CFD에서의 역할

연속 방정식은 CFD 솔버가 **반드시** 만족시켜야 하는 조건입니다:

1. 속도장을 계산할 때 연속 방정식 체크
2. 만족 안 하면 → 압력 보정
3. 반복해서 연속 방정식 만족시킴

이걸 **압력-속도 커플링**이라고 하며, SIMPLE, PISO 같은 알고리즘이 이 역할을 합니다.
